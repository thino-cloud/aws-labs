# aws-labs

Hands-on AWS labs documenting my journey into cloud infrastructure engineering.  
Each lab focuses on a core AWS service — built, tested and documented from scratch.

---

## Labs

### Networking
| Lab | Description | Status |
|-----|-------------|--------|
| VPC Setup | Custom VPC with public/private subnets, route tables and internet gateway | ✅ Done |
| Security Groups | Inbound/outbound rules for EC2 and RDS access control | ✅ Done |
| VPC Peering | Connecting two VPCs across the same region | 🔄 In Progress |

### Compute
| Lab | Description | Status |
|-----|-------------|--------|
| EC2 Deployment | Launch, connect and configure a Linux EC2 instance | ✅ Done |
| EC2 User Data | Automate setup scripts on instance launch | ✅ Done |
| Auto Scaling | Scale EC2 instances based on CloudWatch alarms | 🔜 Planned |

### Storage
| Lab | Description | Status |
|-----|-------------|--------|
| S3 Buckets | Create buckets, manage permissions and enable versioning | ✅ Done |
| S3 Static Website | Host a static site via S3 and CloudFront | 🔜 Planned |

### Security & Identity
| Lab | Description | Status |
|-----|-------------|--------|
| IAM Users & Groups | Least-privilege access with policies and MFA | ✅ Done |
| IAM Roles | Assign roles to EC2 instances for S3 and CloudWatch access | ✅ Done |

### Monitoring
| Lab | Description | Status |
|-----|-------------|--------|
